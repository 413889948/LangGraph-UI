## 为什么

当前 LangGraph 可视化编辑器的用户体验存在三个核心问题：

1. **保存体验差**：每次保存都触发浏览器下载 JSON 文件，文件堆积在下载目录，无法自动保存，用户需要手动管理文件名，存在数据丢失风险。

2. **无撤销功能**：用户误操作后无法恢复，仅支持 Delete/Backspace 删除快捷键，无 undo/redo 历史管理，操作风险高，体验差。

3. **节点命名不友好**：使用 UUID 前缀命名节点（如 `function_a1b2c3d4`），难以记忆和识别，导出的 Python 代码可读性差。

这些问题导致编辑器无法达到现代 Web 应用的 UX 标准，影响用户生产力和使用体验。

## 变更内容

- **新增 localStorage 自动保存**：替代文件下载，实现编辑后 1 秒自动保存，页面加载自动恢复，配额错误降级处理。
- **新增撤销/重做系统**：支持 Ctrl+Z / Ctrl+Shift+Z 快捷键，20 步历史记录，支持所有 mutation 类型（增删改节点/边/Schema）。
- **改进节点命名**：从 UUID 前缀改为语义化递增命名（`fun_1`, `tool_1`, `subgraph_1`），提高代码导出可读性。
- **改进代码生成**：修改边和入口点生成逻辑，使用节点名称而非 ID，确保导出代码无 UUID。

## 功能 (Capabilities)

### 新增功能

- `auto-save`: localStorage 自动保存机制，支持 debounce 保存、配额错误处理、页面加载恢复
- `undo-redo`: 撤销/重做历史管理系统，支持 20 步历史、快捷键绑定、选择状态恢复
- `semantic-naming`: 语义化节点命名系统，按类型递增计数，导出代码无 UUID

### 修改功能

- `code-generation`: 修改代码生成器使用节点名称而非 ID，确保边连接和入口点使用语义化名称
- `node-creation`: 修改节点创建逻辑使用计数器生成语义化名称

## 影响

### 代码影响

- **状态管理**：`src/store/useEditorStore.ts` 扩展支持命名计数器、历史栈、自动保存
- **画布组件**：`src/components/GraphCanvas.tsx` 修改节点创建逻辑，添加撤销/重做快捷键
- **文件操作**：`src/components/FileOperations.tsx` 移除下载逻辑，改用 localStorage
- **代码生成**：`src/utils/codeGenerator.ts` 添加 ID→Name 映射

### 用户体验影响

- **自动保存**：用户无需手动保存，防止数据丢失
- **撤销重做**：误操作可恢复，降低使用焦虑
- **命名可读**：导出代码更易理解和维护

### 技术约束

- localStorage 限制 5-10MB，需配额检测和降级
- 历史栈仅内存存储，页面刷新后清空
- 命名计数器不回收删除编号，避免冲突

### 不影响

- 现有节点 ID 仍为 UUID（保持唯一性）
- 手动导入/导出功能保留（兼容性）
- 多项目管理（范围外）
