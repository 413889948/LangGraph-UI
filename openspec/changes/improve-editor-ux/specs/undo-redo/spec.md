# 撤销/重做规范

## 新增需求

### 需求:撤销操作快捷键

系统必须支持 Ctrl+Z（macOS: Cmd+Z）快捷键撤销上一步操作。

#### 场景:撤销节点创建
- **当** 用户创建一个节点后按 Ctrl+Z
- **那么** 最新创建的节点被删除

#### 场景:撤销节点删除
- **当** 用户删除一个节点后按 Ctrl+Z
- **那么** 被删除的节点恢复

#### 场景:撤销节点移动
- **当** 用户移动节点位置后按 Ctrl+Z
- **那么** 节点恢复到移动前的位置

#### 场景:撤销边连接
- **当** 用户连接两个节点后按 Ctrl+Z
- **那么** 最新创建的边被删除

#### 场景:撤销状态字段编辑
- **当** 用户修改状态 Schema 字段后按 Ctrl+Z
- **那么** 字段恢复到修改前的状态

---

### 需求:重做操作快捷键

系统必须支持 Ctrl+Shift+Z 或 Ctrl+Y（macOS: Cmd+Shift+Z）快捷键重做操作。

#### 场景:重做已撤销的节点创建
- **当** 用户撤销节点创建后按 Ctrl+Shift+Z
- **那么** 被撤销的节点重新出现

#### 场景:重做已撤销的节点删除
- **当** 用户撤销节点删除后按 Ctrl+Shift+Z
- **那么** 节点再次被删除

#### 场景:新操作清空重做栈
- **当** 用户撤销后执行新的编辑操作
- **那么** 重做栈被清空，无法再重做

---

### 需求:历史记录限制

系统必须限制历史栈最多保存 20 步操作。

#### 场景:历史记录达到上限
- **当** 用户执行超过 20 步操作
- **那么** 最旧的历史记录被丢弃

#### 场景:撤销 20 步后无更多历史
- **当** 用户连续撤销 20 次
- **那么** 再次按 Ctrl+Z 无效果

---

### 需求:选择状态恢复

系统必须在撤销节点创建时清除选择状态，避免引用已删除节点。

#### 场景:撤销选中节点的创建
- **当** 用户创建节点后选中它，然后按 Ctrl+Z
- **那么** 节点消失且选择状态清空

#### 场景:撤销删除后不自动选中
- **当** 用户删除选中节点后按 Ctrl+Z
- **那么** 节点恢复但选择状态清空

---

### 需求:快捷键输入保护

系统必须在输入框聚焦时不触发快捷键，保护文本输入。

#### 场景:输入框中 Ctrl+Z 不触发图形撤销
- **当** 用户在节点名称输入框中按 Ctrl+Z
- **那么** 触发输入框的撤销，不触发图形撤销

#### 场景:输入框中 Delete 不删除节点
- **当** 用户在输入框中按 Delete
- **那么** 删除输入框文本，不删除选中节点

#### 场景:textarea 中快捷键保护
- **当** 用户在 textarea 中按快捷键
- **那么** 触发 textarea 默认行为

---

### 需求:历史栈在加载/新建时清空

系统必须在加载新项目或新建项目时清空历史栈，避免跨项目撤销污染。

#### 场景:加载项目后历史清空
- **当** 用户加载一个 JSON 项目文件
- **那么** 历史栈被清空，无法撤销到之前项目

#### 场景:新建项目后历史清空
- **当** 用户点击"新建"按钮
- **那么** 历史栈被清空

---

### 需求:深拷贝历史状态

系统必须深拷贝图形状态到历史栈，避免引用共享导致状态损坏。

#### 场景:撤销恢复完整状态
- **当** 用户创建节点、修改属性后撤销
- **那么** 图形恢复到创建前的完整状态

#### 场景:连续撤销状态一致
- **当** 用户连续撤销 10 次
- **那么** 每次撤销后图形状态正确
