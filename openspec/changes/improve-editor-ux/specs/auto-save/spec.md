# 自动保存规范

## 新增需求

### 需求:编辑后自动保存

系统必须在用户编辑图形后 1 秒自动保存到 localStorage，无需用户手动触发。

#### 场景:创建节点后自动保存
- **当** 用户拖拽创建一个新节点到画布
- **那么** 系统在 1 秒后自动保存图形到 localStorage

#### 场景:移动节点后自动保存
- **当** 用户拖拽移动节点位置
- **那么** 系统在 1 秒后自动保存更新后的图形

#### 场景:删除节点后自动保存
- **当** 用户删除一个节点
- **那么** 系统在 1 秒后自动保存更新后的图形

#### 场景:快速连续编辑防抖
- **当** 用户在 1 秒内连续创建 5 个节点
- **那么** 系统仅在最后一次编辑后 1 秒触发一次保存

---

### 需求:页面加载自动恢复

系统必须在页面加载时自动从 localStorage 恢复上次编辑的图形。

#### 场景:页面刷新后恢复
- **当** 用户刷新浏览器页面
- **那么** 系统从 localStorage 读取并恢复上次保存的图形

#### 场景:首次访问无数据
- **当** 用户首次访问编辑器（localStorage 无数据）
- **那么** 系统显示空画布（默认项目）

#### 场景:浏览器关闭后重新打开
- **当** 用户关闭浏览器后重新打开编辑器
- **那么** 系统恢复上次保存的图形

---

### 需求:存储配额错误处理

系统必须在 localStorage 配额满时显示警告但不阻止用户继续编辑。

#### 场景:存储空间已满
- **当** localStorage 空间不足导致保存失败
- **那么** 系统显示警告 "存储空间已满，自动保存已禁用"
- **且** 用户可以继续编辑图形

#### 场景:警告仅显示一次
- **当** 存储配额错误发生
- **那么** 警告在该会话中仅显示一次

#### 场景:isDirty 状态正确
- **当** 自动保存失败后用户继续编辑
- **那么** isDirty 标记保持为 true

---

### 需求:存储版本管理

系统必须在保存时包含版本号，用于未来迁移。

#### 场景:保存时包含版本
- **当** 系统保存图形到 localStorage
- **那么** 存储结构包含 `version: "1.0.0"` 字段

#### 场景:版本不匹配处理
- **当** 加载的图形版本与当前应用不兼容
- **那么** 系统显示警告并使用空项目

---

### 需求:isDirty 语义定义

isDirty 标记必须仅表示"未成功保存到 localStorage"。

#### 场景:编辑后 isDirty 为 true
- **当** 用户编辑图形
- **那么** isDirty 立即设置为 true

#### 场景:自动保存成功后 isDirty 为 false
- **当** 自动保存成功完成
- **那么** isDirty 设置为 false

#### 场景:自动保存失败后 isDirty 保持 true
- **当** 自动保存失败（如配额满）
- **那么** isDirty 保持为 true

---

### 需求:选择状态不持久化

系统必须不持久化选择状态（selectedNodeId、selectedElementId）到 localStorage。

#### 场景:刷新后选择清空
- **当** 用户选中节点后刷新页面
- **那么** 页面恢复后无节点被选中

#### 场景:视口状态可选持久化
- **当** 用户拖拽/缩放画布后保存
- **那么** 视口状态（x, y, zoom）可以保存到 localStorage

---

### 需求：手动保存快捷键

系统必须支持 Ctrl+S（macOS: Cmd+S）快捷键立即保存图形。

#### 场景:Ctrl+S 手动保存
- **当** 用户按 Ctrl+S
- **那么** 系统立即保存图形到 localStorage
- **且** 显示保存成功提示

#### 场景：手动保存跳过 debounce
- **当** 用户按 Ctrl+S
- **那么** 系统跳过 debounce 直接保存
