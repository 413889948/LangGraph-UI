## 新增需求

### 需求：系统可以导出 StateGraph 定义代码
系统必须将可视化图转换为 LangGraph 的 StateGraph Python 代码，包含完整的图结构定义。

#### 场景：生成 StateGraph 初始化代码
- **当** 用户点击"导出代码"按钮
- **那么** 系统生成包含 StateGraph 初始化和状态 Schema 定义的 Python 代码

#### 场景：生成节点注册代码
- **当** 可视化图中包含已配置的节点
- **那么** 系统为每个节点生成对应的 add_node 调用代码

#### 场景：生成边连接代码
- **当** 可视化图中存在节点间的边连接
- **那么** 系统为每条边生成 add_edge 或 add_conditional_edges 调用代码

### 需求：系统可以生成节点处理函数代码
系统必须为每个可视化节点生成对应的 Python 处理函数定义。

#### 场景：生成预定义模板代码
- **当** 节点使用预定义的处理逻辑模板
- **那么** 系统生成完整的函数代码包含模板逻辑

#### 场景：生成自定义代码
- **当** 用户在配置中输入自定义处理代码
- **那么** 系统将该代码原样包含在导出的 Python 文件中

#### 场景：生成函数签名
- **当** 节点配置了输入输出参数
- **那么** 系统生成包含正确参数类型注解的函数签名

### 需求：系统可以生成状态 Schema 代码
系统必须将可视化状态 Schema 转换为 LangGraph 兼容的 TypedDict 或数据类定义。

#### 场景：生成 TypedDict 定义
- **当** 用户选择 TypedDict 作为 Schema 格式
- **那么** 系统生成包含所有字段和类型注解的 TypedDict 类定义

#### 场景：生成字段类型注解
- **当** 状态字段配置了类型和 Reducer
- **那么** 系统生成正确的类型注解和 Annotated 标记

#### 场景：生成默认值初始化
- **当** 状态字段配置了默认值
- **那么** 系统在代码中包含默认值定义

### 需求：代码导出必须可执行
系统生成的 Python 代码必须是完整可执行的，包含所有必要的导入和依赖。

#### 场景：包含必要导入
- **当** 系统导出代码
- **那么** 代码文件头部包含所有必需的 import 语句（如 StateGraph、TypedDict 等）

#### 场景：生成完整文件结构
- **当** 用户导出项目代码
- **那么** 系统生成可独立运行的 Python 文件，包含 main 函数或图编译调用

#### 场景：代码语法验证
- **当** 用户请求导出代码
- **那么** 系统在导出前验证生成的 Python 代码语法正确性
