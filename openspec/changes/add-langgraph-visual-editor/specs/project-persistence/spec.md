## 新增需求

### 需求：用户可以保存项目到本地文件
系统必须提供项目保存功能，将可视化图结构和配置导出为 JSON 格式文件。

#### 场景：保存新项目
- **当** 用户首次点击"保存项目"并选择保存位置
- **那么** 系统将项目数据导出为 JSON 文件并保存到指定位置

#### 场景：保存已有项目
- **当** 用户点击"保存"按钮修改已打开的项目
- **那么** 系统覆盖原有项目文件，保留文件名和位置

#### 场景：另存为新文件
- **当** 用户选择"另存为"并指定新文件名
- **那么** 系统将项目保存为新文件，原文件保持不变

### 需求：用户可以加载已有项目文件
系统必须提供项目加载功能，从 JSON 文件导入可视化图结构和配置。

#### 场景：加载有效项目文件
- **当** 用户选择"打开项目"并选择一个有效的 JSON 项目文件
- **那么** 系统在画布上还原图的完整结构和所有配置

#### 场景：加载后显示项目名称
- **当** 项目成功加载
- **那么** 系统在界面标题栏显示项目名称

#### 场景：加载时清空当前画布
- **当** 用户加载新项目时当前画布有未保存的修改
- **那么** 系统提示用户保存当前项目，确认后清空画布并加载新项目

### 需求：项目文件必须包含完整的图数据
系统保存的项目 JSON 文件必须包含还原可视化图所需的所有信息。

#### 场景：保存节点数据
- **当** 系统保存项目
- **那么** JSON 文件包含所有节点的位置、类型、配置参数

#### 场景：保存边连接数据
- **当** 系统保存项目
- **那么** JSON 文件包含所有边的源节点、目标节点和连接类型

#### 场景：保存状态 Schema
- **当** 系统保存项目
- **那么** JSON 文件包含完整状态 Schema 定义，包括字段类型和 Reducer 配置

### 需求：系统必须验证项目文件格式
系统必须验证加载的项目文件格式正确性和数据完整性。

#### 场景：检测无效 JSON 格式
- **当** 用户尝试加载格式错误的 JSON 文件
- **那么** 系统显示错误提示并拒绝加载

#### 场景：检测缺失必填字段
- **当** 用户加载缺少必填字段（如节点列表、状态 Schema）的项目文件
- **那么** 系统显示具体的缺失信息并拒绝加载

#### 场景：版本兼容性检查
- **当** 用户加载由不同版本编辑器创建的项目文件
- **那么** 系统检查版本兼容性并在不兼容时给出警告
