## 新增需求

### 需求:用户可以添加和删除状态字段
系统必须提供状态字段的增删操作，支持用户定义 LangGraph 状态对象的属性结构。

#### 场景:添加新字段
- **当** 用户点击"添加字段"按钮并输入字段名称
- **那么** 系统在状态 Schema 中创建一个新字段，显示在字段列表中

#### 场景:删除字段
- **当** 用户点击字段旁的删除按钮并确认
- **那么** 系统从状态 Schema 中移除该字段定义

#### 场景:重命名字段
- **当** 用户编辑字段的名称
- **那么** 系统更新字段名称并验证名称不与其他字段重复

### 需求:用户可以配置字段类型
系统必须支持为每个状态字段选择数据类型，包括基础类型和复合类型。

#### 场景:选择基础类型
- **当** 用户从类型下拉菜单中选择基础类型（如 str、int、bool、float）
- **那么** 系统保存该类型信息并在字段列表中显示

#### 场景:选择复合类型
- **当** 用户选择列表（List）、字典（Dict）或可选（Optional）等复合类型
- **那么** 系统提示用户配置复合类型的参数（如元素类型、键值类型）

### 需求:用户可以配置字段的 Reducer
系统必须允许用户为字段选择预定义的 Reducer 函数，用于定义状态合并逻辑。不支持自定义 Reducer 函数编辑。

#### 场景：选择预定义 Reducer
- **当** 用户从 Reducer 下拉菜单中选择预定义选项（append、replace、sum、max、min）
- **那么** 系统保存该 Reducer 配置并在代码生成时应用


#### 场景:移除 Reducer
- **当** 用户清除字段的 Reducer 配置
- **那么** 系统使用默认覆盖行为处理该字段

### 需求:用户可以设置字段的默认值
系统必须支持为状态字段定义默认值，用于初始化状态对象。

#### 场景:设置简单默认值
- **当** 用户为字段输入默认值（如字符串、数字、布尔值）
- **那么** 系统保存该默认值并在生成代码时应用

#### 场景:设置复杂默认值
- **当** 用户为列表或字典类型输入 JSON 格式的默认值
- **那么** 系统验证 JSON 格式并保存默认值

### 需求:状态编辑器必须验证 Schema 完整性
系统必须提供状态 Schema 的验证功能，确保定义完整且无冲突。

#### 场景:验证必填字段
- **当** 用户尝试保存缺少必填信息（如字段名、类型）的 Schema
- **那么** 系统显示错误提示并阻止保存

#### 场景:检测命名冲突
- **当** 用户创建的字段名与已有字段重复
- **那么** 系统显示冲突警告并要求修改名称
