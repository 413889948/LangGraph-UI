## 上下文

LangGraph 可视化编辑器是一个基于 React + Zustand + React Flow 的 Web 应用，用于可视化构建 LangGraph 工作流。当前仅支持单向代码生成：画布操作 → Python 代码显示。

**当前架构**:
- `GraphDocument` 为核心数据结构，包含 nodes, edges, stateSchema
- `useEditorStore` (Zustand) 管理全局状态
- `codeGenerator.ts` 实现 GraphDocument → Python 代码转换
- `CodePreview.tsx` 只读显示生成的代码

**约束**:
- 浏览器端执行，无后端服务
- 用户代码安全风险需明确声明
- Pyodide 加载时间约 2-3 秒

**利益相关者**: 编辑器用户、LangGraph 开发者

## 目标 / 非目标

**目标：**
- 实现 Python 代码 → GraphDocument 的解析能力
- 提供可编辑的代码面板，支持手动同步到画布
- 确保节点名称唯一性，保证代码解析的可靠性
- 统一代码生成和解析的入口契约（`build_graph()`）

**非目标：**
- 自动实时同步（每次按键执行）
- 支持任意 Python 代码（仅限 LangGraph 核心子集）
- 保留用户函数体/注释的 round-trip 格式化
- Web Worker 隔离执行（后续优化）
- 动态构图支持（循环生成节点）

## 决策

### 1. 解析技术：Pyodide + StateGraph 录制器

**选择**: 使用 Pyodide 执行用户代码，配合 StateGraph 录制器（shim）提取图结构。

**理由**:
- 录制器方案无需安装真实 LangGraph，依赖更小
- 执行结果更确定，错误更可解释
- 避免了 LangGraph 在 Pyodide 中的兼容性问题

**替代方案**:
- ❌ 安装真实 LangGraph：体积大、兼容性差、加载慢
- ❌ 纯 AST 解析：无法处理动态图构建、条件路由等

### 2. 入口契约：`build_graph()` 函数

**选择**: 用户代码必须提供 `build_graph()` 函数，返回 StateGraph 或编译后的图。

**理由**:
- 明确的入口点，避免全局副作用
- 便于错误定位和提示
- 生成器和解析器使用相同契约，确保双向兼容

### 3. 同步策略：手动触发

**选择**: 用户点击"应用到画布"按钮后才执行代码解析和画布更新。

**理由**:
- 避免每次按键都执行 Pyodide 导致的性能问题
- 用户有明确的预期和可控的时机
- 防止循环更新（canvas ↔ code ping-pong）

### 4. 节点匹配：名称 + 唯一性约束

**选择**: 使用节点名称作为匹配键，并强制节点名唯一。

**理由**:
- Python 代码中无节点 ID，只能依赖名称匹配
- 唯一性约束确保匹配的可靠性
- 位置合并时能正确保留用户布局

**替代方案**:
- ❌ 自动重命名：用户体验差，不可预测
- ❌ 允许重名：导入时位置错配，行为不确定

### 5. 安全模型：主线程执行 + UI 声明

**选择**: 在主线程执行用户代码，在 UI 中明确安全声明。

**理由**:
- 简化实现，无需 Web Worker 或 iframe
- 明确告知用户仅支持受信任代码
- 适合本地离线使用的场景

**替代方案**:
- ❌ Web Worker 隔离：实现复杂，通信开销
- ❌ iframe 隔离：同源策略限制

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| Pyodide 加载时间影响用户体验 | 启动时预加载，显示加载进度 |
| 主线程执行可能冻结 UI | 添加安全声明，建议短代码片段 |
| 用户代码可访问 DOM/Fetch | 明确"仅支持受信任代码"声明 |
| 节点名修改导致位置丢失 | 提示用户节点名变更影响布局 |
| 动态构图无法解析 | 显示"不支持动态构图"错误提示 |
| 录制器 API 不完整 | 明确支持的 API 子集，拒绝不支持的调用 |

## 数据流

```
┌─────────────┐                      ┌─────────────┐
│   画布      │  generateLangGraph   │  代码编辑器  │
│  (React Flow)│ ──────────────────────▶│  (textarea) │
│             │                       │             │
│             │ ◀──────────────────────│             │
└─────────────┘   parseCode (手动触发) └─────────────┘
      │                                      │
      ▼                                      ▼
┌─────────────┐                      ┌─────────────┐
│ useEditorStore │                     │ PyodideService │
│             │                       │ + StateGraph   │
│ updateFromCode │ ◀────────────────────│ shim (录制器) │
└─────────────┘                      └─────────────┘
```

## 模块职责

| 模块 | 职责 |
|------|------|
| `pyodideService.ts` | Pyodide 加载、StateGraph shim、代码解析 |
| `CodeEditor.tsx` | 可编辑代码面板、语法高亮、同步按钮、错误显示 |
| `useEditorStore.ts` | 状态管理、`updateFromCode` 原子操作、位置合并 |
| `codeGenerator.ts` | 代码生成，输出 `build_graph()` 格式 |
| `NodeConfigPanel.tsx` | 节点名唯一性验证 |
