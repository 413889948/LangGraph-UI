## 新增需求

### 需求:代码编辑器可编辑
系统必须提供可编辑的 Python 代码输入区域，用户可以修改生成的代码或输入新代码。

#### 场景:用户编辑代码
- **当** 用户切换到代码标签页
- **那么** 系统显示可编辑的代码文本区域，用户可以输入和修改代码

#### 场景:代码显示初始内容
- **当** 画布上有节点存在
- **那么** 代码编辑器显示当前生成的 Python 代码

### 需求:代码语法高亮
系统必须对编辑器中的 Python 代码提供语法高亮显示。

#### 场景:语法高亮显示
- **当** 用户在代码编辑器中输入或查看代码
- **那么** 关键字、字符串、注释等语法元素以不同颜色显示

### 需求:应用到画布按钮
系统必须提供"应用到画布"按钮，用户点击后解析代码并更新画布。

#### 场景:点击应用到画布
- **当** 用户点击"应用到画布"按钮
- **那么** 系统尝试解析当前编辑器中的代码

#### 场景:按钮禁用状态
- **当** Pyodide 服务未加载完成
- **那么** "应用到画布"按钮显示加载状态并禁用

### 需求:解析成功更新画布
当代码解析成功时，系统必须更新画布显示解析结果中的节点和边。

#### 场景:解析成功
- **当** 用户输入有效的 `build_graph()` 代码并点击"应用到画布"
- **那么** 画布显示代码中定义的所有节点和边
- **那么** 节点位置按名称匹配保留（如匹配失败则自动布局）

### 需求:解析失败显示错误
当代码解析失败时，系统必须显示错误信息，并保留原画布状态不变。

#### 场景:语法错误
- **当** 用户输入包含语法错误的代码并点击"应用到画布"
- **那么** 系统显示错误信息（包含行号和错误描述）
- **那么** 画布保持原有状态不变

#### 场景:缺少入口函数
- **当** 用户代码不包含 `build_graph()` 函数
- **那么** 系统显示"请定义 build_graph() 函数"错误提示
- **那么** 画布保持原有状态不变

### 需求:复制代码功能保留
系统必须保留现有的复制代码功能。

#### 场景:复制代码
- **当** 用户点击"复制代码"按钮
- **那么** 当前编辑器中的代码被复制到剪贴板
- **那么** 显示"已复制"提示

### 需求:安全声明显示
系统必须在编辑器区域显示安全声明，告知用户代码执行的风险。

#### 场景:安全声明可见
- **当** 用户查看代码编辑器
- **那么** 显示"仅支持受信任代码"的安全提示

### 需求:编辑器内容持久化
系统必须将编辑器内容存储到 Store 状态中。

#### 场景:内容状态存储
- **当** 用户编辑代码
- **那么** 代码内容存储在 `codeEditorContent` 状态中
