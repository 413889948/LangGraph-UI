## 新增需求

### 需求:节点名唯一性验证
系统必须在节点名输入时验证名称的唯一性，禁止保存重名节点。

#### 场景:重名输入阻止
- **当** 用户在节点配置面板中输入已存在的节点名
- **那么** 系统显示"节点名称已存在"错误提示
- **那么** 保存按钮禁用或输入无法提交

#### 场景:唯一名称允许
- **当** 用户输入唯一的节点名
- **那么** 系统允许保存该名称

### 需求:唯一性验证时机
系统必须在以下时机验证节点名唯一性：

#### 场景:创建新节点时验证
- **当** 用户创建新节点并输入名称
- **那么** 系统检查名称是否与现有节点重复

#### 场景:修改节点名时验证
- **当** 用户修改现有节点的名称
- **那么** 系统检查名称是否与其他节点重复（排除自身）

### 需求:错误提示本地化
系统必须提供中英文的错误提示信息。

#### 场景:中文错误提示
- **当** 用户界面语言为中文且输入重名
- **那么** 显示"节点名称已存在"

#### 场景:英文错误提示
- **当** 用户界面语言为英文且输入重名
- **那么** 显示"Node name already exists"

### 需求:空名称处理
系统必须禁止节点名为空。

#### 场景:空名称阻止
- **当** 用户清空节点名或输入仅包含空白字符
- **那么** 系统显示"节点名称不能为空"错误提示
- **那么** 阻止保存

### 需求:名称格式验证
系统应该验证节点名符合 Python 标识符规范。

#### 场景:非法字符警告
- **当** 用户输入包含特殊字符（如空格、符号）的节点名
- **那么** 系统显示警告提示"建议使用字母、数字和下划线"
- **那么** 允许保存但显示警告
