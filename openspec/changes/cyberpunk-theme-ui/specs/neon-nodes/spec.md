# Neon Nodes Specification

## ADDED Requirements

### 需求:Function 节点必须显示霓虹蓝发光效果

Function 类型的节点必须使用霓虹蓝（#00D4FF）作为边框颜色，并显示发光效果。

#### 场景:Function 节点渲染
- **当** 用户创建一个 Function 类型节点
- **那么** 节点必须显示：
  - 边框颜色：#00D4FF
  - 发光效果：box-shadow 使用 rgba(0, 212, 255, 0.3)

#### 场景:Function 节点悬停效果
- **当** 用户悬停在 Function 节点上
- **那么** 发光强度必须增加 50%

### 需求:Tool 节点必须显示霓虹绿发光效果

Tool 类型的节点必须使用霓虹绿（#22C55E）作为边框颜色，并显示发光效果。

#### 场景:Tool 节点渲染
- **当** 用户创建一个 Tool 类型节点
- **那么** 节点必须显示：
  - 边框颜色：#22C55E
  - 发光效果：box-shadow 使用 rgba(34, 197, 94, 0.3)

### 需求:Subgraph 节点必须显示霓虹紫发光效果

Subgraph 类型的节点必须使用霓虹紫（#A855F7）作为边框颜色，并显示发光效果。

#### 场景:Subgraph 节点渲染
- **当** 用户创建一个 Subgraph 类型节点
- **那么** 节点必须显示：
  - 边框颜色：#A855F7
  - 发光效果：box-shadow 使用 rgba(168, 85, 247, 0.3)

### 需求:节点必须有独特的形状或图标区分

每种节点类型必须通过形状、图标或边框样式进行区分，以支持色盲用户。

#### 场景:节点形状区分
- **当** 用户查看节点
- **那么** 每种节点类型必须有不同的视觉特征：
  - Function: 圆角矩形 + 函数图标
  - Tool: 圆角矩形 + 工具图标
  - Subgraph: 虚线边框 + 子图图标

### 需求:选中节点必须有视觉反馈

当节点被选中时，必须有明显的视觉反馈。

#### 场景:节点选中效果
- **当** 用户选中一个节点
- **那么** 节点必须显示：
  - 发光强度增加
  - 边框宽度增加（可选）

### 需求:节点文字对比度必须满足可访问性标准

节点内的文字必须与节点背景有足够的对比度。

#### 场景:节点文字对比度验证
- **当** 使用 Chrome DevTools Accessibility 面板检查节点内文字
- **那么** 对比度必须大于 4.5:1
