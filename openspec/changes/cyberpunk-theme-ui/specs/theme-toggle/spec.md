# Theme Toggle Specification

## ADDED Requirements

### 需求:系统必须提供主题切换按钮

应用必须提供一个可视化的主题切换按钮，允许用户在深色和浅色主题之间切换。

#### 场景:主题切换按钮渲染
- **当** 用户打开应用
- **那么** Header 区域必须显示主题切换按钮

#### 场景:主题切换功能
- **当** 用户点击主题切换按钮
- **那么** 应用主题必须在 "dark" 和 "light" 之间切换

### 需求:主题选择必须持久化

用户的主题选择必须保存到 localStorage，并在页面刷新后恢复。

#### 场景:主题持久化
- **当** 用户切换到浅色主题并刷新页面
- **那么** 应用必须保持浅色主题

#### 场景:首次访问默认主题
- **当** 用户首次访问应用且 localStorage 中无主题设置
- **那么** 应用必须默认使用深色主题

### 需求:主题切换必须有平滑过渡

主题切换时，所有颜色变化必须有平滑的过渡动画。

#### 场景:主题切换过渡动画
- **当** 用户切换主题
- **那么** 所有颜色变化必须在 200-300ms 内完成过渡

### 需求:主题切换按钮必须有可访问性支持

主题切换按钮必须支持键盘操作和屏幕阅读器。

#### 场景:键盘操作
- **当** 用户使用 Tab 键导航到主题切换按钮
- **那么** 按钮必须显示清晰的焦点环

#### 场景:屏幕阅读器支持
- **当** 屏幕阅读器读取主题切换按钮
- **那么** 必须朗读当前主题状态和按钮功能

### 需求:主题状态必须存储在 Zustand store

主题状态必须作为 Zustand store 的一部分进行管理。

#### 场景:Store 主题状态
- **当** 应用初始化
- **那么** store 必须包含 `theme` 状态和 `setTheme` 方法
