# .sisyphus/ 目录指南

工作流编排目录，用于代理规划、执行跟踪和证据收集。

## 目录结构

```
.sisyphus/
├── plans/          # 只读工作计划（快照）
├── notepads/       # 仅追加的累积智能
├── drafts/         # 进行中的工件
├── evidence/       # 支持证据文件
├── *.log           # 执行日志（工件）
└── *.png           # 截图（工件）
```

## 规则

### plans/（只读）
- 会话开始时创建计划文档
- **禁止**在创建后修改计划内容
- 计划是预期工作的快照，不是动态文档

### notepads/（仅追加）
- 跨会话的累积记忆
- **始终**追加新条目，禁止覆盖
- 按会话/主题组织（例如 `init-deep/`）
- 包含：学习、问题、决策、难题

### drafts/, evidence/, logs, screenshots
- **仅限工件** – 不是实现文件
- 不要基于这些文件建模产品代码
- 截图和日志是证据，不是源代码

## 隐藏文件夹警告

`.sisyphus/` 是包含工作流状态的隐藏目录，不是产品源代码。协作工作时不要忽略此目录 – 它包含关键的规划和记忆。

## 相关文件

- 根目录 `AGENTS.md` – 项目概述和命令
- `openspec/AGENTS.md` – OpenSpec 变更工作流
- `src/AGENTS.md` – 应用代码约定
